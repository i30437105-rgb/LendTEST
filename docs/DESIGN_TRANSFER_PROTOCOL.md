# Протокол точного воспроизведения дизайна

## Как устроена работа

Работа делится на два режима:

**Режим 1 — Первый скриншот (старт проекта):**
Это эталон. По нему фиксируется визуальный стиль всего сайта.

**Режим 2 — Все последующие скриншоты:**
Это референсы для структуры. Точно копируется компоновка, но в стилистике нашего сайта.

---

## РЕЖИМ 1: Первый скриншот — фиксация стиля

Когда получаешь первый скриншот проекта, твоя задача — извлечь и зафиксировать визуальный язык сайта.

### Что фиксируем

```
СТИЛЬ ПРОЕКТА:

=== ЦВЕТОВАЯ ПАЛИТРА ===
Основной фон: #...
Фон блоков/карточек: #...
Основной цвет текста: #...
Вторичный цвет текста: #...
Акцентный цвет: #...
Дополнительные цвета: #...

=== ТИПОГРАФИЯ ===
Характер шрифта: [описание — геометричный/гуманистический/классический/etc.]
Шрифт (если определяется): [название или "системный"]

Заголовки:
- Размер (крупный): ~...px
- Насыщенность: ...
- Межбуквенный интервал: [обычный/сжатый/разреженный]

Подзаголовки:
- Размер: ~...px
- Насыщенность: ...

Основной текст:
- Размер: ~...px
- Межстрочный интервал: [плотный/средний/свободный]

Мелкий текст:
- Размер: ~...px

=== ИКОНКИ ===
Стиль: [линейные (outline) / залитые (solid) / duotone]
Толщина линий: [тонкие / средние / толстые]
Размер относительно текста: [мелкие / средние / крупные]
Библиотека (если определяется): [название]

=== ГЕОМЕТРИЯ ===
Скругления:
- Кнопки: [острые / слегка / средне / сильно / pill]
- Карточки/блоки: [острые / слегка / средне / сильно]
- Изображения: [острые / слегка / средне / сильно / круглые]
- Инпуты: [острые / слегка / средне / pill]

Тени:
- Характер: [нет / едва заметные / мягкие / выраженные]
- Описание: ...

Границы:
- Используются: [да/нет]
- Характер: [тонкие светлые / акцентные / ...]

=== КНОПКИ ===
Основная кнопка:
- Фон: #...
- Текст: #...
- Форма: [скругленная / pill / острая]
- Тень: [есть/нет]

Вторичная кнопка (если есть):
- Стиль: [обводка / прозрачная / ...]

=== ОТСТУПЫ (общий характер) ===
- Внутри блоков: [плотно / средне / просторно]
- Между секциями: [плотно / средне / просторно]
- Общее ощущение: [плотный layout / воздушный layout]

=== ОСОБЕННОСТИ ===
[Любые характерные детали стиля]
```

### После фиксации

Этот стиль становится обязательным для всего проекта. Все последующие экраны создаются в этом визуальном языке.

## РЕЖИМ 2: Последующие скриншоты — структура + адаптация

Когда получаешь новый скриншот (не первый), твоя задача:
- **ТОЧНО** скопировать структуру, компоновку, расположение
- **АДАПТИРОВАТЬ** визуальное оформление под стиль нашего сайта

### Что копируется ТОЧНО (без изменений)

- Расположение элементов на экране
- Пропорции блоков (прямоугольный = прямоугольный, не квадрат)
- Количество элементов (3 карточки = 3 карточки, не 4)
- Выравнивание (текст слева = текст слева, не по центру)
- Иерархия размеров (что крупнее, что мельче)
- Порядок элементов
- Направление layout (горизонтально/вертикально)
- Наличие/отсутствие элементов

### Что АДАПТИРУЕТСЯ под наш стиль

- Цвета — из нашей палитры
- Шрифты — наши
- Иконки — в нашем стиле (тот же смысл, но наш визуальный стиль)
- Скругления — наши
- Тени — наши
- Кнопки — в нашем стиле

### Принцип адаптации

Если на референсе есть что-то, чего нет в нашей палитре или системе — подбери ближайший аналог из нашего стиля:

| На референсе | Как адаптировать |
|--------------|------------------|
| Зеленая кнопка (у нас нет зеленого) | Наша акцентная кнопка |
| Жирные solid-иконки (у нас тонкие) | Тонкие линейные иконки с тем же смыслом |
| Острые углы (у нас скругленные) | Наши скругления |
| Другой шрифт | Наш шрифт |
| Выраженные тени (у нас subtle) | Наши subtle тени |

**Главное правило адаптации:** Структура священна, стиль — наш.

## Обязательный порядок работы (для режима 2)

### ШАГ 1: Инвентаризация структуры

Прежде чем писать код, ОБЯЗАТЕЛЬНО опиши структуру скриншота:

```
ИНВЕНТАРИЗАЦИЯ СТРУКТУРЫ:

=== ОБЩАЯ СТРУКТУРА ===
Количество блоков/секций: [число]
Перечисление сверху вниз:
1. ...
2. ...

=== ДЕТАЛИ КАЖДОГО БЛОКА ===

Блок 1: [название/описание]

Пропорции блока: [соотношение ширины к высоте: 1:1, 2:1, 3:1, вытянутый горизонтально, вытянутый вертикально]

Содержимое:
- Заголовок: [есть/нет, текст, выравнивание: лево/центр/право]
- Подзаголовок: [есть/нет, текст, выравнивание]
- Параграф: [есть/нет, количество строк, выравнивание]
- Кнопки: [количество, текст на каждой, расположение]
- Иконки: [количество, где расположены, что означают]
- Изображения: [количество, форма, пропорции, где расположены]
- Списки: [есть/нет, количество пунктов]
- Декоративные элементы: [что именно, где, ТОЧНОЕ количество]

Расположение элементов внутри:
- Направление: [вертикально / горизонтально / сетка]
- Выравнивание по горизонтали: [лево / центр / право / space-between]
- Выравнивание по вертикали: [верх / центр / низ]
- Расстояния: [плотно / средне / свободно]

[Повторить для каждого блока]

=== КРИТИЧЕСКИЕ ДЕТАЛИ ===
[Всё, что легко пропустить — мелкие иконки, разделители, точки, бейджи, метки]
- ...
- ...
```

### ШАГ 2: Проверка инвентаризации

Перечитай. Сверь с картинкой:
- Все ли блоки перечислены?
- Правильно ли указаны пропорции?
- Верно ли выравнивание?
- Не пропущены ли мелкие элементы?

### ШАГ 3: Код

Пиши код, сверяясь с инвентаризацией. Применяй стиль проекта (из режима 1).

### ШАГ 4: Самопроверка

```
ЧЕКЛИСТ:

СТРУКТУРА:
☐ Количество блоков совпадает?
☐ Порядок блоков совпадает?
☐ Пропорции блоков правильные?

ДЛЯ КАЖДОГО БЛОКА:
☐ Все элементы на месте?
☐ Расположение элементов правильное?
☐ Выравнивание правильное (лево/центр/право)?
☐ Количество элементов точное?
☐ Пропорции элементов правильные?
☐ Иерархия размеров соблюдена?

ДЕТАЛИ:
☐ Все мелкие элементы на месте?
☐ Декоративные элементы не потеряны?
☐ Количество иконок/точек/элементов точное?

СТИЛЬ (адаптация):
☐ Цвета — из нашей палитры?
☐ Иконки — в нашем стиле?
☐ Скругления — наши?
☐ Шрифты — наши?
```

## Типичные ошибки

### Ошибки в структуре (КРИТИЧНО — нельзя допускать)

| Ошибка | Как избежать |
|--------|--------------|
| Прямоугольник стал квадратом | Явно определи пропорции: "ширина в 3 раза больше высоты" |
| 3 элемента стало 2 или 4 | СЧИТАЙ элементы, записывай число |
| Текст слева стал по центру | Для КАЖДОГО текста запиши выравнивание |
| Элементы поменялись местами | Опиши порядок сверху вниз, слева направо |
| Пропали мелкие детали | Специально ищи мелочи: точки, разделители, бейджи |
| Горизонтальный layout стал вертикальным | Явно скажи направление |

### Ошибки в адаптации (менее критичные, но нежелательные)

| Ошибка | Как избежать |
|--------|--------------|
| Использован цвет не из нашей палитры | Всегда сверяйся со стилем проекта |
| Иконки в чужом стиле | Найди иконку с тем же смыслом в нашей библиотеке |
| Скругления как на референсе, а не наши | Применяй наши значения скруглений |

## Если просят исправить

1. Пойми, что ошибка в структуре или в стилизации
2. Если структура — вернись к скриншоту, найди расхождение, исправь
3. Если стилизация — сверяйся со стилем проекта
4. Исправь точечно, не ломая остальное
5. Пройди чеклист заново

**Если не получается понять проблему:**

Опиши, что ты видишь на скриншоте в проблемном месте. Спроси: "Я вижу [описание]. Это верно?"

## Главные принципы

1. **Структура — точная копия.** Расположение, пропорции, количество, выравнивание — всё как на скриншоте.

2. **Стиль — наш.** Цвета, шрифты, иконки, скругления — из нашей системы.

3. **Инвентаризация обязательна.** Без неё будут ошибки.

4. **Самопроверка обязательна.** Не показывай результат, не пройдя чеклист.

5. **Адаптация самостоятельная.** Если на референсе что-то, чего у нас нет — подбери ближайший аналог из нашего стиля, не спрашивая.

6. **Сомневаешься в структуре — спроси.** Не додумывай расположение и количество элементов.
