## Паттерны ошибок (НЕ допускать)

| № | Проблема | Решение | Железное правило |
|---|----------|---------|------------------|
| 1 | **Inline SVG вместо библиотеки** | Все иконки из Lucide, хранятся в `/assets/icons/`, подключаются через `<img>` | НИКОГДА не рисовать SVG вручную path-ами |
| 2 | **Заголовки font-weight 700** | Использовать weight 600 (semibold) | Для заголовков: weight 600, НЕ 700 |
| 3 | **Дескриптор с italic** | font-style: normal | Курсив НЕ использовать для описательных текстов |
| 4 | **Line-height дескриптора завышен** | line-height 1.4 | Для body/дескрипторов: 1.4, максимум 1.6 |
| 5 | **Белый навбар на светлом фоне** | background: transparent, border: none | Навбар сливается с фоном секции |
| 6 | **Двойной отступ между секциями** | Верхняя секция даёт padding-bottom, нижняя — padding-top: 0 | При одинаковом фоне НЕ допускать стыка двух padding. Десктоп: 150px, мобайл: 80px |
| 7 | **min-height: 100vh создаёт непредсказуемые отступы** | Убрать min-height, использовать явный padding | НЕ использовать min-height: 100vh для контроля отступов. Всегда фиксированные padding |
| 8 | **Разные border-radius и padding для карточек** | border-radius карточки = 16px, изображений = 12px, padding = 12px | ЖЕЛЕЗНОЕ ПРАВИЛО карточек: 16px / 12px / 12px |
| 9 | **margin-bottom на последнем элементе секции** | margin-bottom: 0 для последнего элемента | Последний элемент секции НЕ должен иметь margin-bottom |
| 10 | **align-self: stretch при flex column на мобайле** | Явно указать `flex: none; align-self: auto; width: auto` | При flex row → column ОБЯЗАТЕЛЬНО переопределить align-self, flex, width |
| 11 | **highlight-lime::before не видна вне Hero** | Добавить `position: relative; z-index: 0` на контейнер заголовка | Вне Hero всегда создавать stacking context для highlight-lime |
