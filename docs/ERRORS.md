## Паттерны ошибок (не допускать повторно)

### 1. Inline SVG вместо файлов из библиотеки
- **Проблема:** иконки и стрелки рисовались вручную path-ами прямо в HTML
- **Решение:** все иконки скачиваются из Lucide и хранятся в `/assets/icons/`, подключаются через `<img>`
- **Правило:** НИКОГДА не рисовать SVG вручную. Всегда использовать библиотеку Lucide

### 2. Заголовки слишком жирные
- **Проблема:** font-weight 700 для заголовков выглядел грубо
- **Решение:** weight 600 (semibold) — более элегантно
- **Правило:** для ClashDisplay заголовков использовать weight 600, не 700

### 3. Дескриптор — italic не нужен
- **Проблема:** курсив на дескрипторе выглядел неуместно
- **Решение:** font-style: normal
- **Правило:** курсив не использовать для описательных текстов

### 4. Line-height дескриптора — не завышать
- **Проблема:** line-height 1.8 давал слишком много воздуха между строками
- **Решение:** line-height 1.4
- **Правило:** для body/дескрипторов — 1.4, максимум 1.6

### 5. Белый навбар на светлом фоне — лишняя полоса
- **Проблема:** белый background навбара создавал визуальную полосу
- **Решение:** background: transparent, border: none
- **Правило:** навбар должен сливаться с фоном секции
