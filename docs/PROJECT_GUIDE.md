## Продукт

Тест для оценки эффективности маркетинга. Результат теста:
- Каждый элемент маркетинга классифицируется в **серую** или **красную** зону
- Пользователь получает **пошаговый план исправления** нарушений
- Цель: рост выручки, повышение эффективности маркетинга

**Что мы делаем:** Посадочная страница (лендинг) для холодного трафика. Цель лендинга — продать идею прохождения теста.

**Целевая аудитория:**
- Собственники микрокомпаний России
- Маленькие бизнесы абсолютно разного профиля
- Таргетируемся ИСКЛЮЧИТЕЛЬНО на собственников

**Технологии:** Чистый HTML / CSS / JS, без фреймворков и конструкторов

## Бренд

- **Название:** MetodZMS
- **Логотип:** иконка crosshair (Lucide) + текст "MetodZMS" шрифтом ClashDisplay 600

## ЖЕЛЕЗНЫЕ ПРАВИЛА (нарушать нельзя)

### 1. Экран за экраном
- Сайт создаётся **последовательно, по одному экрану**
- НИКОГДА не создавать весь сайт целиком за раз
- Порядок: создаём экран → утверждаем → переходим к следующему
- Дизайн-концепция следующего экрана базируется на утверждённом предыдущем

### 2. Референсы от заказчика и единая стилистика
- **Экран 1 (Hero) задаёт стилистику всего сайта** — цвета, шрифты, отступы, визуальный язык
- Заказчик присылает дизайн-референс для каждого последующего экрана
- Референс = инструкция по **структуре и компоновке** (layout, расположение блоков, иконок, распределение по экрану)
- **Визуальный стиль** (цвета, типографика, скругления, тени, эстетика) — ВСЕГДА берётся из утверждённого первого экрана
- Весь сайт должен быть в **единой гармоничной стилистике**
- Никакой самодеятельности в дизайне без согласования
- **ОБЯЗАТЕЛЬНО:** работа со скриншотами ведётся по протоколу DESIGN_TRANSFER_PROTOCOL.md

### 3. Запрет на создание/рисование графики
- НЕ создавать/рисовать самостоятельно: иконки, стрелки, фотографии, скриншоты, картинки, рисунки, видео
- НЕ использовать inline SVG, нарисованные вручную в коде
- Иконки и стрелки — скачивать из открытых библиотек и сохранять в `/assets/icons/`
- Фотографии, скриншоты, картинки — поставляет заказчик
- Визуальные файлы хранятся в папке `/assets/`

**Полная типографика:** см. PROJECT_STYLE.md

### Типографика — иерархия уровней

| Уровень | Шрифт | Десктоп | Мобайл | Weight | Цвет |
|---------|-------|---------|--------|--------|------|
| H1 (Hero) | Unbounded | 55px | 28px | 600 | #1a1a2e |
| H2 (секции) | Unbounded | 22px | 17px | 400 | #1a1a2e |
| Body | Manrope | 18px | 14px | 400 | #7a7f8a |

### Правила типографики
- **НЕ использовать font-weight 700** на заголовки
- **НЕ использовать italic** на описательные тексты
- **Стиль надзаголовка:** normal (НЕ italic)
- **Стиль подзаголовка:** normal (НЕ italic)

### Отступы и сетка

**Десктоп:**
- Боковые отступы Hero: 144px
- Межсекционные отступы: 150px
- Максимальная ширина контента: 1320px
- Базовая единица spacing: 8px

**Мобайл (<768px):**
- Боковые отступы: 20px
- Межсекционные отступы: 80px

### Правило межсекционных отступов (КРИТИЧНО)
> Секции с одинаковым фоном НЕ ДОЛЖНЫ двойным стыком padding увеличивать расстояние.
>
> **Принцип:** верхняя секция отвечает за расстояние через свой `padding-bottom`. Нижняя секция ставит `padding-top: 0`.
>
> **Последний элемент контента** внутри секции НЕ должен иметь `margin-bottom` — иначе он искажает расстояние между секциями.

### Кнопки
- **CTA (primary):** фон #1a1a2e, текст белый, Manrope 17px/600
- Паддинг: 24px 36px
- Скругление: 16px
- Иконка: zap (молния, Lucide), заливка, белая, 22px
- **Hover:** фон #c8f542, текст и иконка → #1a1a2e, glow-тень, scale(1.04)

### Карточки — Тип 1: Контентные карточки

**Геометрия (ЖЕЛЕЗНЫЕ ПАРАМЕТРЫ):**
- Border-radius карточки: 16px
- Border-radius изображений/контента внутри: 12px
- Padding базовый: 12px
- Gap между элементами внутри: 12px

**Стиль:**
- Фон: #ffffff / #fafafa / #c8f542
- Тень: 0 25px 60px rgba(0,0,0,0.10), 0 6px 20px rgba(0,0,0,0.05)
- Hover: translateY(-6px), усиление тени

### Иконки
- **Библиотека:** Lucide (v0.563.0, ISC лицензия)
- **Стиль:** rounded stroke, stroke-width 2
- **Хранение:** `/assets/icons/` — SVG-файлы
- **Подключение:** через `<img src="assets/icons/...">`, НЕ inline SVG

### Адаптивность
- Мобайл: < 768px
- Планшет: 768–1024px
- Десктоп: > 1024px
- Mobile-first подход
- **Мобильную версию корректировать отдельно** — не масштабировать десктопные значения автоматически

### Анимации — сдержанные, плавные, живые
- **Hover кнопок:** scale(1.04) + shadow, 0.25s ease
- **Hover карточек:** translateY(-6px) + усиление тени
- **Счётчик:** пульсация scale(1.3), 0.3s cubic-bezier
- **Без агрессивных эффектов** (bounce, shake, flash)
- **@media (prefers-reduced-motion: reduce)** — обязательно

**Детали анимаций:** см. PROJECT_STYLE.md
