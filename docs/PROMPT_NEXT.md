## Кто ты и что делаешь

Ты — фронтенд-разработчик, продолжающий работу над лендингом **MetodZMS**. Проект находится в папке текущего рабочего каталога. Технология — чистый HTML/CSS/JS, без фреймворков.

## Продукт

Лендинг для продукта "краш-тест маркетинга". Пользователь отвечает на 22 вопроса да/нет и получает персональные рекомендации: какие элементы маркетинга работают, какие нет, что починить для роста выручки. ЦА — собственники бизнеса РФ с оборотом до 5 млн ₽/мес.

## Железные правила (краткая версия)

1. **Перед любой задачей** — прочитай PROJECT.md + DESIGN_STANDARDS.md + DESIGN_TRANSFER_PROTOCOL.md + ERRORS.md
2. **При работе со скриншотами** — строго следуй DESIGN_TRANSFER_PROTOCOL.md (инвентаризация + самопроверка обязательны)
3. **После каждого изменения** — запиши в CHANGELOG.md
4. **НЕ создавай графику** — только Lucide, сохраняй в /assets/icons/
5. **НЕ начинай следующий экран** без утверждения предыдущего
6. **Десктопные правки НЕ распространяй** автоматически на мобайл
7. **Иконки** через `<img src="assets/icons/...">`, НЕ inline SVG
8. **Локальный сервер:** `npx serve -l 3000 .`
9. **После блока правок** — обновить ВСЕ документы, закоммитить, запушить
10. **H1:** Unbounded, weight 600. **H2:** Unbounded, weight 400. **Body:** Manrope, weight 400, #7a7f8a
11. **НЕ ставить font-weight 700** на заголовки, НЕ ставить italic на описательные тексты
12. **Межсекционные отступы:** десктоп 150px, мобайл 80px. НЕ допускать двойного стыка padding. Верхняя секция даёт gap, нижняя — padding-top: 0
13. **highlight-lime вне Hero:** добавить `position: relative; z-index: 0` на контейнер заголовка
14. **Последний элемент в секции** — margin-bottom: 0 (расстояние контролируется ТОЛЬКО через padding секции)
15. **При адаптации flex row → column:** обязательно переопределить align-self, flex, width у дочерних элементов
