# WORKFLOW — Порядок работы

## Процесс создания каждого экрана

```
1. Заказчик присылает тексты для экрана → сохраняем в COPY.md
2. Заказчик присылает дизайн-референс (скриншот + ссылку на сайт-источник)
3. Следуем DESIGN_TRANSFER_PROTOCOL.md:
   - Режим 1 (первый экран): фиксируем стиль проекта
   - Режим 2 (все последующие): инвентаризация структуры + адаптация под наш стиль
4. Делаем ТОЧНУЮ КОПИЮ структуры референса (с оригинальными текстами!)
5. Заказчик оценивает точность воспроизведения
6. Правки по точности → фиксируем в ERRORS.md → вносим
7. После утверждения копии — подставляем тексты заказчика из COPY.md
8. При подстановке русских текстов — проверяем кириллицу в шрифтах, заменяем если нужно
9. Финальные правки → утверждение экрана → переходим к следующему
```

**Правило:** следующий экран начинается ТОЛЬКО после утверждения предыдущего.

**ОБЯЗАТЕЛЬНО:** при работе со скриншотами строго следовать протоколу DESIGN_TRANSFER_PROTOCOL.md.

## Правила работы

1. **Перед началом любой задачи** — прочитать PROJECT_GUIDE.md + DESIGN_TRANSFER_PROTOCOL.md
2. **После каждого изменения** — записать в CHANGELOG.md
3. **При получении замечания** — записать в ERRORS.md, проверить паттерны ошибок
4. **Не начинать следующий экран** без утверждения предыдущего
5. **Не создавать графику** — скачивать из Lucide, сохранять в /assets/icons/
6. **Сначала точная копия референса** — потом замена текстов
7. **Запускать локальный сервер** для просмотра (npx serve -l 3000 .)
8. **Новые экраны** — ОБЯЗАТЕЛЬНО наследуют стиль из PROJECT_STYLE.md
9. **Десктопные правки** не распространять автоматически на мобайл
10. **После завершения блока правок** — обновить ВСЕ документы, закоммитить и запушить на GitHub
