# WORKFLOW — Порядок работы

## Процесс создания каждого экрана

```
1. Заказчик присылает тексты для экрана → сохраняем в COPY.md
2. Заказчик присылает дизайн-референс → анализируем
3. Воспроизводим референс с контентом заказчика (HTML/CSS/JS)
4. Заказчик смотрит результат
5. Правки → фиксируем в ERRORS.md → вносим
6. Утверждение экрана → переходим к следующему
```

**Правило:** следующий экран начинается ТОЛЬКО после утверждения предыдущего.

## Правила работы
1. **Перед началом любой задачи** — прочитать PROJECT.md
2. **После каждого изменения** — записать в CHANGELOG.md
3. **При получении замечания** — записать в ERRORS.md, исправить, отметить как решённое
4. **Не начинать следующий экран** без утверждения предыдущего
5. **Не создавать графику** — только использовать из /assets/
